# Day 2 – Service vs Product, Conditionals & Complexity

## Topics Covered
- Difference between service-based and product-based companies
- Basic differences between C and Python
- Conditional statements (if, elif, else)
- Introduction to time complexity and space complexity

## Key Notes
- Service-based companies: work on client projects, often focus on delivery, maintenance, support.
- Product-based companies: build their own products, focus on innovation, quality, scalability.
- C vs Python:
  - C is compiled, lower-level, faster, and closer to hardware.
  - Python is interpreted, high-level, easier to write, slower but more productive.
- Conditional statements control flow based on conditions:
  - Syntax:
    ```python
    if condition:
        # block
    elif another_condition:
        # block
    else:
        # default block
    ```
- Time complexity describes how the runtime grows with input size (Big-O notation like O(1), O(n), O(n^2)).
- Space complexity describes how memory usage grows with input size.

## Example Concepts / Pseudo-Code
- Program to check grade based on marks using multiple `elif` conditions.
- Program to compare two numbers and print which is greater.
- Simple example of O(1) vs O(n): accessing one element vs iterating over all elements in a list.

## What I Practiced
- Writing if/elif/else logic for decision-making (e.g., pass/fail, age classification).
- Using logical operators (`and`, `or`, `not`) in conditions.
- Thinking about how loops and operations relate to time complexity.
